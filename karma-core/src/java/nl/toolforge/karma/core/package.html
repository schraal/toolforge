The core package contains packages and classes related to the very core of the Karma toolset (you'd never guess that, right ?).

The core of the Karma toolset consists of the architecture to manage so-called manifests (see the {@link nl.toolforge.karma.core.Manifest} interface), groups of modules that make up an application or part thereof.

A developer can control a manifest using a set of commands. Each module in a manifest is - obviously - located in a version control system. Support for CVS has been implemented, support for Subversion is planned (that's the most diplomatic answer, isn't it ?).
